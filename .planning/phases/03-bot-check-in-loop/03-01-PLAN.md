<task type="auto">
  <name>Task 1: Create check-in conversation flow with inline keyboard</name>
  <files>src/handlers/checkin.ts, src/services/checkin-sender.ts</files>
  <action>
Create src/handlers/checkin.ts — the callback query handler for the multi-step energy rating flow:

1. The check-in flow uses grammY inline keyboard with callback_data. The flow walks through 4 energy types sequentially:
   - Step 1: Show "Физическая энергия" with buttons 1-10 (callback_data: "checkin:{logType}:physical:{value}")
   - Step 2: After physical rated, edit message to show "Ментальная энергия" buttons
   - Step 3: After mental rated, show "Эмоциональная энергия" buttons
   - Step 4: After emotional rated, show "Духовная энергия" buttons
   - Step 5: After all 4 rated, save EnergyLog to DB, edit message to show summary

2. Use an in-memory Map<number, PendingCheckIn> keyed by telegramId to track partial check-in state (which energies have been rated so far, what logType). This is fine for MVP with 20-50 users. The PendingCheckIn type: `{ logType: string; physical?: number; mental?: number; emotional?: number; spiritual?: number }`.

3. Export `handleCheckinCallback(ctx: CallbackQueryContext<Context>)` — parses callback_data, updates pending state, shows next energy or saves to DB when complete.

4. Export `sendCheckInMessage(chatId: number, logType: "morning" | "evening")` — sends initial check-in message with first energy keyboard. Morning message: "Доброе утро! Как твои энергии сегодня? Оцени каждую от 1 до 10." Evening message: "Добрый вечер! Подведём итоги дня. Как твои энергии сейчас?"

5. Inline keyboard layout: two rows of 5 buttons each (1-5 on first row, 6-10 on second row).

6. When all 4 energies rated, create EnergyLog via prisma with the user's DB id (look up user by telegramId from ctx.from.id). Edit the message to show: "Записал! Физическая: {N}, Ментальная: {N}, Эмоциональная: {N}, Духовная: {N}".

7. Use energy type labels in Russian: Физическая, Ментальная, Эмоциональная, Духовная.

Create src/services/checkin-sender.ts:
1. Import bot from "../bot.js" and prisma from "../db.js"
2. Export `async function sendCheckInToAll(logType: "morning" | "evening")` — queries all users from DB, calls sendCheckInMessage for each user's telegramId (convert BigInt to Number for chatId). Wrap each send in try/catch to handle blocked/deleted users gracefully (log warning, continue to next user).

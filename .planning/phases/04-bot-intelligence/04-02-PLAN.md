<task type="auto" tdd="true">
  <name>Task 1: Create recommendation engine service</name>
  <files>src/services/recommendations.ts, src/services/recommendations.test.ts</files>
  <behavior>
    - getRecommendations returns 1-2 recovery practices from knowledge base for each detected drop
    - Practices returned match the dropped energy type (no cross-type substitution)
    - When AI personalization is available, the recommendation includes a personalized context string
    - When AI fails, the raw knowledge base recommendation is still returned (graceful degradation)
    - If no drops detected but lowestEnergy exists, returns a maintenance tip for that type
    - If no data at all, returns empty array
  </behavior>
  <action>
    Create `src/services/recommendations.ts` exporting:

    1. **Types:**
    ```typescript
    interface Recommendation {
      energyType: EnergyType;
      practice: Practice;
      personalizedTip: string;   // AI-personalized or fallback to practice.description
      reason: string;            // Why this was recommended (e.g., "Физическая энергия упала на 3 пункта")
    }
    ```

    2. **`getRecommendations(diagnosticResult: DiagnosticResult, userId: number): Promise<Recommendation[]>`**
    - For each drop in `diagnosticResult.drops`:
      - Call `getRecoveryPractices(drop.energyType)` from knowledge base
      - Pick top 1-2 practices (first entries are highest priority)
      - Build a context string with the user's recent history (query last 5 logs)
      - Call AI to personalize: pass the practice name+description + user context, ask AI to give a SHORT (2-3 sentence) personalized tip explaining WHY this practice matters for this user RIGHT NOW
      - System prompt for AI: "Ты персонализируешь рекомендацию из базы знаний. НЕ придумывай новые практики. Объясни КРАТКО (2-3 предложения) почему именно эта практика важна для пользователя сейчас, учитывая его историю энергии."
      - If AI call fails, use practice.description as fallback
    - For chronicLows not already covered by drops: add 1 practice each
    - If no drops and no chronicLows but lowestEnergy exists: add 1 maintenance practice for that type
    - Cap total recommendations at 3 (avoid overwhelming the user)

    3. **`formatRecommendations(recommendations: Recommendation[]): string`**
    - Format as readable Russian text
    - Each recommendation: energy type label, reason for recommendation, practice name, personalized tip
    - If empty: return empty string (caller decides whether to show anything)

    For testing: mock both the AI service and knowledge base. Test that:
    - Correct practices are selected for each energy type
    - AI failure doesn't break the flow
    - Cap of 3 recommendations is enforced
    - No cross-type practices appear
  </action>

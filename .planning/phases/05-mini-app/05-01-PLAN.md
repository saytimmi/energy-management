<task type="auto">
  <name>Task 1: HTTP server with static serving and dashboard API</name>
  <files>src/server.ts, src/api/dashboard.ts, src/index.ts, package.json</files>
  <action>
1. Add `express` and `@types/express` to dependencies:
   - `npm install express` and `npm install -D @types/express`
   - Express is the simplest choice — no need for Fastify's performance for 20-50 users

2. Create `src/api/dashboard.ts`:
   - Export a function `dashboardRoute` that takes Express Router
   - GET `/api/dashboard` endpoint:
     - Accepts `?telegramId=XXXXX` query param
     - Looks up user by telegramId (BigInt)
     - Finds latest EnergyLog for that user (orderBy createdAt desc, take 1)
     - Returns JSON: `{ physical: number, mental: number, emotional: number, spiritual: number, loggedAt: string }` or `{ error: "no_data" }` if no logs exist
     - Returns 404 if user not found

3. Create `src/server.ts`:
   - Export `startServer(port?: number)` and `stopServer()`
   - Create Express app
   - Serve static files from `public/` directory (use `path.resolve` relative to project root, not dist)
   - Mount API routes under `/api`
   - Add CORS header: `Access-Control-Allow-Origin: *` (needed for Telegram WebView)
   - Default port: `process.env.PORT || 3000`
   - Return the http.Server instance from startServer for clean shutdown

4. Update `src/index.ts`:
   - Import `startServer` and `stopServer` from `./server.js`
   - Call `startServer()` after bot starts, log the port
   - Add `stopServer()` to SIGINT/SIGTERM handlers before prisma disconnect
   - Add `config.port` to `src/config.ts`: `port: parseInt(process.env.PORT || "3000", 10)`
  </action>

   - Export `historyRoute(router: express.Router)` following same pattern as dashboardRoute
   - GET `/api/history` endpoint:
     - Query params: `telegramId` (required), `period` ("week" | "month", default "week")
     - Look up user by telegramId
     - Calculate date range: week = 7 days ago, month = 30 days ago
     - Query EnergyLog entries within range, ordered by createdAt asc
     - Return JSON array: `{ date: string (ISO), physical: number, mental: number, emotional: number, spiritual: number }[]`
     - If multiple logs per day, average them (group by date)
     - Return 404 if user not found, empty array if no logs

2. Register route in `src/server.ts`:
   - Import historyRoute
   - Mount on the router alongside existing dashboard route
  </action>

   - Create Chart.js line chart with:
     - X axis: dates (formatted as day/month, e.g., "15/03")
     - Y axis: energy level 0-10

   - Period toggle handler: clicking "Неделя"/"Месяц" refetches with period=week/month and updates chart

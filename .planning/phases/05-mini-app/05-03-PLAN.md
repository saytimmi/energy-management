     - Look up user by telegramId

     - If fewer than 3 logs, return `{ hasEnoughData: false, message: "Нужно минимум 3 записи энергии для анализа паттернов" }`
     - Build a structured summary of the energy data for AI:
       - Format each log as: `{date}: Физ={physical}, Мент={mental}, Эмоц={emotional}, Дух={spiritual}`
       - Include basic stats: averages per type, min/max per type, day-of-week averages
     - Call `askAI()` with a prompt asking for pattern analysis:
       - System context: "Ты анализируешь паттерны энергии пользователя за последний месяц. Выдели 3-5 конкретных паттернов. Каждый паттерн должен ссылаться на реальные данные. Формат: нумерованный список. Не давай общих советов — только наблюдения о паттернах."
       - User message: the formatted energy data summary
     - Return JSON: `{ hasEnoughData: true, insights: string, stats: { avgPhysical, avgMental, avgEmotional, avgSpiritual, totalLogs, periodDays } }`
     - Wrap AI call in try/catch — on failure return stats without insights: `{ hasEnoughData: true, insights: null, stats: {...}, error: "ai_unavailable" }`

   - Import analyticsRoute

# Telegram Bot -- Команды, AI Tool Use, Потоки

## Команды бота

| Команда | Описание | Доступ |
|---------|----------|--------|
| `/start` | Главное меню. Новый пользователь -- приветствие + объяснение 4 типов энергии. Возвращающийся -- AI-приветствие на основе истории. Inline-кнопки: "Записать энергию", "Мой отчёт", "Energy App". | Все |
| `/energy` | Запуск чекина энергии (InlineKeyboard 1-10 по 4 типам). Синоним кнопки "Записать энергию". | Все |
| `/report` | Анализ энергии за последние дни: диагностика + рекомендации. Формат HTML. | Все |
| `/help` | Справка: список команд, описание 4 типов энергии, расписание чекинов (9:00 / 21:00). | Все |
| `/kaizen` | Диагностика системы: ошибки за 24ч, перформанс API, статистика пользователей. `/kaizen improve` -- запуск GitHub Action для автоулучшения кода. | Только владелец |

Все остальные текстовые сообщения обрабатываются AI через буфер (3 сек для одиночного, 8 сек для серии). Голосовые -- транскрибируются через Groq Whisper и обрабатываются как текст.

## Потоки разговоров

### Energy Checkin (severity-based)

1. Бот отправляет InlineKeyboard 1-10 для "Физическая энергия"
2. Пользователь нажимает оценку, бот показывает следующий тип (ментальная, эмоциональная, духовная)
3. После 4 оценок -- сохранение в DB, slot-to-slot сравнение с предыдущим чекином того же типа (утро с утром)
4. Severity-анализ каждого типа:
   - **Critical** (drop >= 4 или level <= 3 + drop >= 3) -- показывает падение + причины + рекомендации
   - **Moderate** (drop 2-3 или level <= 3 + drop >= 1) -- показывает падение + 1-2 рекомендации
   - **Improved** (rise >= 2) -- "Что помогло?"
   - **Stable** -- "Стабильно"
5. При падении/росте -- мульти-выбор триггеров (кнопки + "Свой вариант")
6. Нажатие "Готово" -- сохранение наблюдений, вопрос о деталях
7. Пользователь описывает ситуацию текстом (или /skip) -- context сохраняется в Observation
8. Вечерний чекин дополнительно показывает intraday-изменения (сравнение с утром)

### Формирование миссии (3 вопроса)

1. Пользователь выражает желание определить миссию
2. AI задаёт ПЕРВЫЙ вопрос: "Кто ты? Не должность и не роль. Что тебя определяет как человека?"
3. Пользователь отвечает -- AI задаёт ВТОРОЙ: "Каково твоё место в мире? Что ты даёшь другим?"
4. Пользователь отвечает -- AI задаёт ТРЕТИЙ: "Что ты оставишь после себя? Какой след?"
5. AI формулирует statement из 3 ответов: "Вот как я это вижу: [statement]. Резонирует?"
6. Только после подтверждения -- вызов `set_mission` с identity, purpose, legacy, statement

### Постановка цели

1. Пользователь говорит о цели (или AI предлагает после balance assessment)
2. AI определяет сферу, горизонт (год/квартал), формулирует конкретно
3. Если есть метрика -- заполняет metric + targetValue
4. Вызов `set_goal`
5. После создания -- AI предлагает создать привычку для этой цели

### Кайдзен-час (рефлексия)

1. Утреннее напоминание (8:00 local) или пользователь инициирует сам
2. AI показывает контекст вчерашнего дня: энергия, привычки, наблюдения
3. Ключевой вопрос: "Что вчера было самым важным?" или "Что бы ты сделал иначе?"
4. Диалог: инсайты, паттерны, выводы
5. Если родился алгоритм -- `save_algorithm`
6. Завершение -- `save_reflection` с summary и insights

### Balance Assessment (AI-guided)

1. Пользователь хочет оценить баланс жизни (или прошло 14 дней)
2. AI вызывает `start_balance_assessment` -- получает данные по каждой сфере
3. По каждой из 8 сфер AI обсуждает аспекты (здоровье: сон, активность, питание, самочувствие, энергия)
4. Обсуждение выливается в оценку -- `rate_life_area` с subScores и assessmentType: "ai_guided"
5. После всех сфер -- предложение выбрать фокус-сферы и установить идентичность

## Правила поведения AI

**Анти-GPT стиль** -- самое важное правило. Бот пишет как друг в телеграме, не как AI:
- Строчные буквы, фрагменты, без точек в конце коротких фраз
- Максимум 2-4 предложения, максимум 1 вопрос за ответ
- Запрещены: маркированные списки, жирный текст, "Отлично!", "Понимаю!", канцелярит
- Умеренные эмодзи, к месту

**Tool-first**: если информации достаточно для действия -- AI вызывает инструмент, не переспрашивает. Никогда не говорит "создал"/"записал" без реального вызова tool.

**DATA-теги**: когда пользователь упоминает состояние энергии (но не просит чекин), AI добавляет в конец ответа: `<!--DATA:{"energyType":"...","direction":"...","trigger":"...","context":"..."}-->` для пассивного сбора данных.

**Фокус**: строго управление энергией и привычками. Не болтовня на посторонние темы. Не личные вопросы.

## Smart Nudges (проактивные уведомления)

Один nudge в день, в 9:00 по локальному времени, выбирается по наивысшему приоритету:

| Приоритет | Тип | Триггер |
|-----------|-----|---------|
| 90 | Goal-Habit Gap | Цель в фокус-сфере, но нет привычек в этой сфере |
| 85 | Low Consistency | Привычки для цели -- consistency < 30% |
| 80 | Energy Pattern | 3+ замеров одного типа энергии <= 4 за последние дни |
| 75 | Streak at Risk | Сильная привычка (strength > 50) потеряет стрик завтра |
| 70 | Milestone | Стрик достиг 7 / 21 / 30 / 60 / 90 / 100 дней |
| 65 | Weekly Goal Review | Понедельник -- inline-кнопки для целей (done/drop) |
| 60 | Balance Drift | Фокус-сфера упала в оценке по сравнению с прошлой |
| 50 | AI Fallback | Haiku генерирует персональный nudge на основе контекста |

Шаблонные nudges = 0 токенов. AI fallback = ~200 токенов Haiku.

## Awareness Gaps (пробелы в данных)

AI получает awareness context при каждом сообщении и может мягко направлять пользователя:

| Приоритет | Тип | Триггер |
|-----------|-----|---------|
| 100 | `no_energy` | Нет ни одного замера энергии |
| 90 | `no_habits` | Нет активных привычек |
| 80 | `no_balance` | Нет оценки баланса, аккаунт > 3 дней |
| 70 | `stale_balance` | Баланс старше 14 дней |
| 60 | `no_mission` | Нет миссии, аккаунт > 14 дней |
| 55 | `no_goals` | Нет целей, аккаунт > 21 дней |
| 52 | `no_reflection` | Вчера не было рефлексии |
| 50 | `empty_meaning` | Привычка без whyToday / whyIdentity |
| 45 | `goal_without_habits` | Цель без привычек в той же lifeArea |
| 40 | `no_focus_areas` | Баланс оценён, но нет фокус-сфер |
| 35 | `low_area_no_goal` | Сфера <= 4/10 без цели |
